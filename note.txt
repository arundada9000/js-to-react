INTRODUCTION
===========================================================

Teaching someone coding is honestly one of the sweetest things ever.
Do not try to make her "memorize JavaScript".
Your goal is to make her:
- comfortable with code
- curious
- confident
- excited to build things

IMPORTANT TEACHING RULES ❤️
-----------------------------------------------------------

1. Never rush.
2. Make her type everything herself.
3. Let her make mistakes.
4. Laugh at bugs together.
5. Build tiny cute projects often.
6. Teach visually.
7. Explain WHY before HOW.
8. Repeat concepts naturally.

BEST LEARNING METHOD:
-----------------------------------------------------------

Step 1:
You explain the concept.

Step 2:
You code one example.

Step 3:
She explains the code back to you.

Step 4:
She builds a similar thing herself.

This is the fastest learning method.

HOW TO TEACH map()
-----------------------------------------------------------

Old way:
loop through array manually.

Then show:
map() automatically creates new array.

Example:
const names = ["Arun","Subekshya"]

const newNames = names.map(name => {
   return name + " ❤️"
})

IMPORTANT TEACHING ADVICE ❤️
===========================================================

WHEN SHE GETS STUCK:
-----------------------------------------------------------

Never immediately give answer.

Instead ask:
- What is the error?
- What do you think is happening?
- Which line looks suspicious?

This develops problem solving.


HOW TO KEEP HER MOTIVATED ❤️
-----------------------------------------------------------

1.
Celebrate tiny wins.

2.
Say:
"That was real developer thinking."

3.
Build things she personally likes.

4.
Keep projects emotional and fun.

5.
Do coding dates.

6.
Do "1 bug challenge" together.

7.
Let her decorate projects creatively.


BIGGEST BEGINNER MISTAKES
-----------------------------------------------------------

- watching too many tutorials
- not coding enough
- memorizing syntax
- fear of errors
- comparing themselves

Teach her:
Errors are NORMAL.
Every developer googles things.
